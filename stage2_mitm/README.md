# 阶段 2：Diffie-Hellman 中间人攻击（MITM）

目标：

- 在客户端与服务端之间充当中间人
- 针对 DH 密钥协商流程实施攻击，使中间人可解密双方传输内容

本目录当前提供：

- 主动篡改 DH 握手，与两端分别协商密钥
- 可解密阶段 1 的 AES-256-GCM 加密内容，并再加密转发（含 rekey）
- 运行时会在中间人终端打印明文的十六进制

## 构建与运行

```bash
make
./bin/mitm_proxy --listen 127.0.0.1 --lport 9001 --target 127.0.0.1 --tport 9000
```

之后让阶段 1 的客户端连到 9001，中间人转发到真实服务端 9000。
